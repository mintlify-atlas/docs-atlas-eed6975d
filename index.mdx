SQL Tools Service is the backend engine powering SQL tooling in [Azure Data Studio](https://github.com/microsoft/azuredatastudio) and the [mssql extension for VS Code](https://marketplace.visualstudio.com/items?itemName=ms-mssql.mssql). It exposes database management features over a JSON-RPC protocol via stdio, making it easy to integrate into any editor or tool.

## Get started

<Steps>
  <Step title="Download the service">
    Download the SQL Tools Service binary for your platform from the [GitHub releases page](https://github.com/Microsoft/sqltoolsservice/releases).
  </Step>
  <Step title="Launch the process">
    Start the service by running `Microsoft.SqlTools.ServiceLayer` and communicating via its stdin/stdout streams using JSON-RPC messages.
  </Step>
  <Step title="Connect to a database">
    Send a `connection/connect` request with your server details to establish a database connection.
  </Step>
  <Step title="Execute queries">
    Use `query/executeString` to run SQL and `query/subset` to page through results.
  </Step>
</Steps>

## Supported platforms

SQL Tools Service runs on Windows, macOS, and Linux. It targets .NET and communicates through stdio, making it embeddable in any application that can spawn a child process.

<Note>
  SQL Tools Service is used by Azure Data Studio and the VS Code mssql extension. If you're building SQL tooling for editors, this service handles the heavy lifting.
</Note>
