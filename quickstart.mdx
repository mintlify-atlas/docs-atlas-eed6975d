title: "Quick Start"
description: "Connect to SQL Server and execute your first query"
---

<Info>
  SQL Tools Service communicates entirely over stdio using JSON-RPC 2.0. You interact with it by writing messages to its stdin and reading responses from its stdout.
</Info>

<Steps>
  <Step title="Download the binaries">
    Download the SQL Tools Service binaries for your platform from the [GitHub releases page](https://github.com/Microsoft/sqltoolsservice/releases).

    Release builds are labeled with a version number (e.g., `v4.x.x Release`). Daily development builds end with `-alpha`.

    Alternatively, you can build from source:

    ```bash
    git clone https://github.com/Microsoft/sqltoolsservice
    cd sqltoolsservice
    dotnet restore
    dotnet build src/Microsoft.SqlTools.ServiceLayer
    ```
  </Step>

  <Step title="Launch the process">
    Start the service by running the `Microsoft.SqlTools.ServiceLayer` executable. Your host application needs to spawn this as a child process and attach to its stdin and stdout streams.

    ```bash
    ./Microsoft.SqlTools.ServiceLayer
    ```

    <Note>
      Treat the stdio streams as byte streams rather than text, since header and body parts of messages may use different encodings. The header is ASCII; the body is UTF-8.
    </Note>

    The service runs one instance per SQL workspace. Launch a new process for each top-level folder or project your user works with.
  </Step>

  <Step title="Send the initialize message">
    Before sending any other requests, initialize the language server session. This follows the [Language Server Protocol](https://github.com/Microsoft/language-server-protocol/blob/master/protocol.md#initialize) `initialize` request.

    **Request:**

    ```
    Content-Length: 78\r\n
    \r\n
    ```

    ```json
    {
      "jsonrpc": "2.0",
      "id": 1,
      "method": "initialize",
      "params": {
        "rootPath": "/home/user/my-sql-project"
      }
    }
    ```

    The service responds with its capabilities, including supported completion trigger characters, formatting support, hover, and definition providers.

    **Response:**

    ```json
    {
      "jsonrpc": "2.0",
      "id": 1,
      "result": {
        "capabilities": {
          "textDocumentSync": 2,
          "completionProvider": {
            "resolveProvider": true,
            "triggerCharacters": [".", ":", "\\", "[", "\""]
          },
          "hoverProvider": true,
          "definitionProvider": true,
          "documentFormattingProvider": true,
          "documentRangeFormattingProvider": true,
          "signatureHelpProvider": {
            "triggerCharacters": [" ", ","]
          }
        }
      }
    }
    ```
  </Step>

  <Step title="Connect to a database">
    Send a `connection/connect` request to establish a database connection. The `ownerUri` is a unique identifier (typically a file path) that binds this connection for use in subsequent requests.

    **Request:**

    ```
    Content-Length: 207\r\n
    \r\n
    ```

    ```json
    {
      "jsonrpc": "2.0",
      "id": 2,
      "method": "connection/connect",
      "params": {
        "ownerUri": "file:///home/user/my-sql-project/query.sql",
        "connection": {
          "serverName": "localhost",
          "databaseName": "AdventureWorks",
          "userName": "sa",
          "password": "YourPassword123"
        }
      }
    }
    ```

    Use `"authenticationType": "Integrated"` for Windows Authentication instead of `userName` and `password`.

    The connection completes asynchronously. The service sends a `connection/complete` notification when the connection is established:

    ```json
    {
      "jsonrpc": "2.0",
      "method": "connection/complete",
      "params": {
        "ownerUri": "file:///home/user/my-sql-project/query.sql",
        "connectionId": "a1b2c3d4-...",
        "connectionSummary": {
          "serverName": "localhost",
          "databaseName": "AdventureWorks",
          "userName": "sa"
        },
        "serverInfo": { ... }
      }
    }
    ```
  </Step>

  <Step title="Execute a query">
    Send a `query/executeString` request to run a SQL statement. Use the same `ownerUri` as the connection.

    **Request:**

    ```
    Content-Length: 153\r\n
    \r\n
    ```

    ```json
    {
      "jsonrpc": "2.0",
      "id": 3,
      "method": "query/executeString",
      "params": {
        "ownerUri": "file:///home/user/my-sql-project/query.sql",
        "query": "SELECT TOP 10 ProductID, Name, ListPrice FROM Production.Product ORDER BY ListPrice DESC"
      }
    }
    ```

    The initial response is empty — it just acknowledges the request was received:

    ```json
    {
      "jsonrpc": "2.0",
      "id": 3,
      "result": {}
    }
    ```

    The service then sends notifications as the query runs. First, `query/message` notifications with row-count updates, then a `query/complete` notification when execution finishes:

    ```json
    {
      "jsonrpc": "2.0",
      "method": "query/complete",
      "params": {
        "ownerUri": "file:///home/user/my-sql-project/query.sql",
        "batchSummaries": [
          {
            "id": 0,
            "hasError": false,
            "executionStart": "2024-01-15T10:30:00.000Z",
            "executionEnd": "2024-01-15T10:30:00.042Z",
            "executionElapsed": "00:00:00.0420000",
            "resultSetSummaries": [
              {
                "id": 0,
                "batchId": 0,
                "rowCount": 10,
                "complete": true,
                "columnInfo": [
                  { "columnName": "ProductID", "dataTypeName": "int" },
                  { "columnName": "Name", "dataTypeName": "nvarchar" },
                  { "columnName": "ListPrice", "dataTypeName": "money" }
                ]
              }
            ]
          }
        ]
      }
    }
    ```
  </Step>

  <Step title="Retrieve results">
    Once you receive `query/complete`, fetch the actual row data with `query/subset`. Use the `batchIndex` and `resultSetIndex` from the `query/complete` summary to identify the result set.

    **Request:**

    ```
    Content-Length: 178\r\n
    \r\n
    ```

    ```json
    {
      "jsonrpc": "2.0",
      "id": 4,
      "method": "query/subset",
      "params": {
        "ownerUri": "file:///home/user/my-sql-project/query.sql",
        "batchIndex": 0,
        "resultSetIndex": 0,
        "rowsStartIndex": 0,
        "rowsCount": 10
      }
    }
    ```

    **Response:**

    ```json
    {
      "jsonrpc": "2.0",
      "id": 4,
      "result": {
        "resultSubset": {
          "rowCount": 10,
          "rows": [
            [
              { "displayValue": "749", "isNull": false },
              { "displayValue": "Road-150 Red, 62", "isNull": false },
              { "displayValue": "3578.27", "isNull": false }
            ],
            [
              { "displayValue": "750", "isNull": false },
              { "displayValue": "Road-150 Red, 44", "isNull": false },
              { "displayValue": "3578.27", "isNull": false }
            ]
          ]
        }
      }
    }
    ```

    Rows are returned as 2D arrays of cell values. Each cell has a `displayValue` string and an `isNull` flag. Use `rowsStartIndex` and `rowsCount` to page through large result sets.
  </Step>
</Steps>
