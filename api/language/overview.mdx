The Language Service provides editor intelligence for SQL files: real-time diagnostics, completions, hover information, signature help, go-to-definition, and find references. It is compatible with the [VS Code Language Server Protocol (LSP)](https://microsoft.github.io/language-server-protocol/), so any LSP-capable editor can integrate with it using standard protocol messages.

## What the language service provides

- **Diagnostics** ‚Äî SQL syntax and semantic errors reported as you type via `textDocument/publishDiagnostics`
- **Completions** ‚Äî keyword, table, column, and schema suggestions via `textDocument/completion`
- **Hover information** ‚Äî type and object documentation on `textDocument/hover`
- **Signature help** ‚Äî function/procedure parameter hints via `textDocument/signatureHelp`
- **Go-to-definition** ‚Äî navigate to a table or object definition via `textDocument/definition`
- **Find references** ‚Äî locate all usages of an object via `textDocument/references`
- **Extension completions** ‚Äî load custom completion assemblies via `completion/extLoad`

## Connection management prerequisite

The language service operates on connection-bound SQL documents. Before language features activate for a file, you must connect that file's `ownerUri` using `connection/connect`. Once connected, the service parses the document and begins providing diagnostics and completions.

## Supported LSP methods

### Lifecycle

| Method | Direction | Description |
|---|---|---|
| `initialize` | Client ‚Üí Service | Initialize the language server and negotiate capabilities |
| `shutdown` | Client ‚Üí Service | Request a graceful shutdown |
| `exit` | Client ‚Üí Service (notification) | Terminate the process after `shutdown` |

### Workspace

| Method | Direction | Description |
|---|---|---|
| `workspace/didChangeConfiguration` | Client ‚Üí Service (notification) | Send updated settings to the service |
| `workspace/didChangeWatchedFiles` | Client ‚Üí Service (notification) | Notify when watched files change on disk |

### Document synchronization

| Method | Direction | Description |
|---|---|---|
| `textDocument/didOpen` | Client ‚Üí Service (notification) | Register a document with the language service |
| `textDocument/didChange` | Client ‚Üí Service (notification) | Send incremental or full content updates |
| `textDocument/didClose` | Client ‚Üí Service (notification) | Unregister a document |
| `textDocument/didSave` | Client ‚Üí Service (notification) | Notify on document save |

### Language features

| Method | Direction | Description |
|---|---|---|
| `textDocument/completion` | Client ‚Üí Service (request) | Get completion items at a cursor position |
| `completionItem/resolve` | Client ‚Üí Service (request) | Fetch full documentation for a completion item |
| `textDocument/hover` | Client ‚Üí Service (request) | Get hover info for the symbol under the cursor |
| `textDocument/signatureHelp` | Client ‚Üí Service (request) | Get function/procedure parameter signatures |
| `textDocument/definition` | Client ‚Üí Service (request) | Navigate to the definition of a symbol |
| `textDocument/references` | Client ‚Üí Service (request) | Find all references to a symbol |

### Diagnostics

| Method | Direction | Description |
|---|---|---|
| `textDocument/publishDiagnostics` | Service ‚Üí Client (notification) | Push SQL parse and semantic errors to the editor |

## Non-LSP extensions

| Method | Direction | Description |
|---|---|---|
| `completion/extLoad` | Client ‚Üí Service (request) | Load an external .NET assembly that contributes custom completion items |
